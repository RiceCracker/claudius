## ADR-005: gosu + setpriv privilege drop
**Status:** Accepted
**Context:** The entrypoint must run as root initially (user-init hook, creating home directory with correct UID). After setup, Claude Code must run as the host user (no root parent remaining).
**Decision:** Use `gosu $HOST_USER` which uses exec internally — replaces itself with the target process, leaving no root parent. With `CLAUDIUS_SUDO=0` (default): additionally `setpriv --no-new-privs` to prevent setuid escalation.
**Consequences:**
+ No root process remains in PID tree after privilege drop
+ Signals go directly to Claude Code
+ `--no-new-privs` prevents setuid binary exploitation (default)
- `CLAUDIUS_SUDO=1` lifts `--no-new-privs` — necessary for passwordless sudo to work; capabilities still bounded
