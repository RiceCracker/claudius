## ADR-006: SNI anti-spoof DNS verify
**Status:** Superseded by ADR-007 (2026-04-28) — proxy removed entirely.
**Context:** An attacker (or injected prompt) could reach a blocked IP by connecting to it while presenting an allowed hostname in the TLS SNI extension. The proxy would allow the connection based on SNI match without checking whether the IP actually belongs to that hostname.
**Decision:** When a TCP connection is allowed *only* because of the SNI/Host header (not because the destination IP itself matches an ACL rule), the proxy resolves the SNI hostname via async DNS and rejects the connection if the IP doesn't match.
**Consequences:**
+ Prevents SNI-spoofing attacks — a significant attack surface for AI agent containment
+ Logged explicitly: `BLOCK tcp SNI-spoof api.anthropic.com → 93.184.216.34`
- Adds DNS round-trip latency per new HTTPS connection (~50–200ms) — accepted for a dev tool
- Fails open on DNS errors (transient DNS failures don't break legitimate traffic)
