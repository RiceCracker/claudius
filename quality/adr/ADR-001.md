## ADR-001: Host-side iptables proxy (no container-side rules)
**Status:** Superseded by ADR-007 (2026-04-28) — proxy removed entirely.
**Context:** Network filtering could be implemented inside the container (container-side iptables) or on the Docker bridge from the host side. Container-side requires NET_ADMIN inside the container. gVisor (runsc) virtualizes the network stack — container-side iptables don't affect the actual host network path.
**Decision:** All iptables/ip6tables rules are installed by the proxy sidecar running with `--network host`. The claudius container has no network filtering of its own.
**Consequences:**
+ Works identically with runc and gVisor — packets traverse the veth pair and bridge regardless of runtime
+ No NET_ADMIN needed in the claudius container
+ Proxy sidecar cleanup removes all rules on exit — no persistent host state
- Requires the proxy sidecar to run with `--network host` (host network namespace access)
