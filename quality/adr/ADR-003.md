## ADR-003: Docker socket proxy (inspect-only default)
**Status:** Accepted
**Context:** Developers want Claude to inspect the host Docker environment (diagnose services, check logs). Mounting `/var/run/docker.sock` directly gives full Docker access including `docker run --privileged`. A filtered proxy (Tecnativa docker-socket-proxy) allows read-only access.
**Decision:** Docker socket access is provided via a docker-socket-proxy sidecar on an isolated internal network. Write operations are blocked at the proxy level by default. `CLAUDIUS_DOCKER_WRITE=1` enables write ops with documented risk.
**Consequences:**
+ Inspect operations always available (safe default)
+ Claude can diagnose running services without risk
- `CLAUDIUS_DOCKER_WRITE=1` + `CLAUDIUS_SUDO=1` = effective host root (documented, user responsibility)
- `docker inspect` exposes `Config.Env` of other containers — database passwords visible (documented limitation)
