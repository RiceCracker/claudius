## ADR-002: gVisor as optional runtime
**Status:** Accepted
**Context:** Strongest isolation without a full VM requires intercepting syscalls at the kernel boundary. gVisor provides a user-space kernel (runsc) that intercepts all syscalls. Trade-offs: performance overhead, Wayland clipboard incompatibility, x86_64/arm64 only.
**Decision:** gVisor is supported as an optional runtime via `CLAUDIUS_RUNTIME=runsc`. Default remains runc. Network filtering is unaffected (host-side proxy intercepts at bridge, below gVisor's netstack).
**Consequences:**
+ Strongest isolation for high-risk scenarios
+ Network proxy still works (intercepts at Docker bridge layer)
- Wayland clipboard not forwarded with gVisor (X11 still works)
- Performance overhead (~20–30% for syscall-heavy workloads)
- Additional install step (`make gvisor-install`)
