## Findings

### Critical (Blocker)
*none*

### Important

- [x] **I-1: `project_name` with `:` breaks Docker volume mount** ‚Äî `claudius.sh:37,159`
  `basename` does not sanitize colons. A project directory named `my:project` produces
  `-v "/real/path:/home/user/my:project"` which Docker parses as three `:` segments
  (invalid). The container fails to start with a confusing Docker error.
  **Fix:** sanitize `project_name` ‚Äî replace `:` with `-` (or fail with a clear message).

- [x] **I-2: Silent DNS failure in `resolve_ip_map`** ‚Äî `entrypoint.py:157-162`
  When a non-wildcard UDP hostname in `CLAUDIUS_ALLOW` fails DNS resolution at startup,
  the hostname is silently dropped from `IP_MAP`. UDP traffic to that host is blocked
  with no log entry. Users get no indication that their ACL entry was ineffective.
  **Fix:** log a `WARN` when `getaddrinfo` fails for a rule entry.

### Optional

- [ ] **O-1: `tests/cases/` directory is empty** ‚Äî `tests/cases/`
  Directory exists but contains no files. The README implies separate test case files
  exist; all tests are inline in `integration.sh`.

- [ ] **O-2: `except Exception: pass` in `parse_sni` / `parse_http_host`** ‚Äî `entrypoint.py:291,301`
  Parse failures are silently swallowed. On malformed input, `host=None` is passed to
  `is_allowed`, which then falls back to IP-only matching. Behaviour is correct but
  a debug-level log would help diagnose unexpected blocks.

- [ ] **O-3: macOS compatibility not documented** ‚Äî `README.md`
  claudius works on macOS with `CLAUDIUS_NO_PROXY=1` (container isolation, Docker socket
  proxy, privilege drop all functional). README doesn't mention this at all.
